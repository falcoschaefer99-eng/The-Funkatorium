# MUSE Brain ŌĆö Glossary

Canonical terms and function additions for launch documentation.

---

## Core terms

| Term | Definition |
|------|------------|
| **Observation** | A memory record with texture (`salience`, `vividness`, `grip`, `charge`, `somatic`) rather than plain text only. |
| **Charge phase** | Processing lifecycle of an observation: `fresh ŌåÆ active ŌåÆ processing ŌåÆ metabolized`. |
| **Paradox loop** | An explicit unresolved tension between identity cores, tracked in `open_loops` and processed intentionally. |
| **Runner contract** | Runtime trigger output describing whether to run, which task to execute, context policy, and prompt contract. |
| **Captured skill** | Review-gated skill artifact emitted from successful runtime execution (`candidate ŌåÆ accepted ŌåÆ degraded ŌåÆ retired`). |
| **Daemon proposal** | Reviewable suggestion generated by daemon loops (`pending` until accepted/rejected). |

---

## Proactivity v1.1 terms

| Term | Definition |
|------|------------|
| **Intention pulse** | Runtime drift scan across stale high-priority tasks, open loops, and stale project next-actions; injected into trigger response and runner prompt. |
| **Recall contract** | Structured "remind me later" rule with due window, scope, and priority; materialized by daemon into task or proposal. |
| **Fact candidate** | Structured productivity fact extracted from context summary (`decision`, `deadline`, `goal`, `preference`, `assignment`). |
| **Fact commitment** | Reviewable proposal created from high-confidence `decision`/`deadline` fact candidates when commitment bridge is enabled. |
| **Commitment bridge** | `mind_context` promotion path from extracted facts into proposal-backed commitments with provenance and thresholds. |
| **Shadow mode** | Diagnostics mode that previews filtering/promotion impact without persisting execution side effects. |

## Added function surface (v1.1)

### `mind_context action=set`

New fields:

- `recall_contracts[]`
- `auto_commit`
- `commitment_mode` (`shadow` \| `proposal`)
- `commitment_threshold`
- `commitment_project_entity_id`

Validation highlights:

- `auto_commit=true` requires `extract_facts=true`
- proposal materialization requires `extraction_mode=write`

### `mind_runtime action=trigger`

New output fields:

- top-level `intention_pulse`
- `runner_contract.intention_pulse`

Prompt contract update:

- autonomous prompt now includes an "Intention pulse" block with summary lines

### `mind_propose`

New proposal types:

- `recall_contract`
- `fact_commitment`

Review behavior:

- accepting either type can materialize an actionable task with provenance metadata

### Daemon orchestration

New daemon pass:

- `recall-contracts` (runs before `task-scheduling`)

Materialization modes:

- `scope=task` ŌåÆ create open task (dedupe by `source=recall_contract:<id>`)
- `scope=proposal` ŌåÆ create pending `recall_contract` proposal
